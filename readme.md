# 新闻挖掘器

一个智能新闻早期识别系统，在新闻事件"萌芽期"捕捉关键信号，提前发现具有重大影响力的现象级新闻。

## 项目亮点

**核心突破**：摒弃传统"事后判定"，采用「多维度信号监测 + 动态阈值升级 + 趋势预判」机制，捕捉新闻事件从"萌芽"到"爆发"的关键信号，解决新冠疫情、ChatGPT 等重大事件"早期被忽视"的问题。

**核心原则**：早期识别不追求 100% 精准，而是"宁筛错不筛漏"，先将高潜力萌芽事件纳入「观察池」持续跟踪，再通过信号强化逐步升级，既保证早期捕捉，又通过后续验证降低误判。

## 核心功能

- **早期识别**：监测六大维度萌芽信号，在事件爆发前捕捉关键特征
- **三级事件池**：观察池 → 候选池 → 确定池，动态升级/降级机制
- **时效展示**：优先展示近三天的大新闻
- **历史归档**：归档历史大新闻，便于回顾
- **人机协同**：通过管理员反馈持续优化识别模型

## 早期识别机制

### 三级事件池体系

将新闻事件按潜在影响力信号强度分为三级池，配合监测周期 + 信号阈值实现自动升级/降级：

| 事件池等级 | 对应阶段    | 监测周期  | 核心特征                     | 展示/处理逻辑                        |
| ---------- | ----------- | --------- | ---------------------------- | ------------------------------------ |
| **观察池** | 萌芽期      | 1小时/次  | 单领域弱信号，小众渠道传播   | 仅管理员可见，系统持续跟踪信号变化   |
| **候选池** | 发酵期      | 30分钟/次 | 多信号强化，主流渠道开始关注 | 管理员重点审核，可选择性推送测试用户 |
| **确定池** | 爆发/定型期 | 1天/次    | 跨领域强信号，全域广泛传播   | 纳入正式新闻列表，推送所有普通用户   |

### 升级/降级规则

- **升级**：满足下一等级信号阈值 + 持续时间，自动升级
  - 观察池事件 24 小时内满足 3 个及以上萌芽信号 → 升级为候选池
  - 候选池事件满足跨领域强信号 → 升级为确定池
- **降级**：连续监测周期内信号持续衰减，自动降级
  - 候选池事件 72 小时内无新信号强化 → 降级为观察池
  - 连续 1 周衰减 → 剔除
- **锁定**：管理员手动标记"高潜力"，锁定为候选池，不受信号衰减影响

### 六大维度萌芽信号

#### 1. 传播渠道信号

重大影响力事件早期多在专业/垂直渠道传播，再向大众渠道渗透。

| 信号类型       | 量化/质性指标                                | 基础阈值                           |
| -------------- | -------------------------------------------- | ---------------------------------- |
| 专业渠道热度   | 医疗/科技/金融等垂直领域专业平台的内容互动量 | 24小时内互动量环比飙升 500%+       |
| 渠道渗透速度   | 从专业渠道向大众社交平台的传播时间差         | 渗透时间 ≤ 48小时                  |
| 小众渠道权威性 | 内容发布者为领域内权威主体                   | 发布者粉丝 ≥ 1万且领域垂直度 ≥ 90% |

#### 2. 数据指标信号

早期事件无全网热度，但核心量化数据出现"非常规增长"。

| 信号类型     | 量化/质性指标                                         | 基础阈值                                |
| ------------ | ----------------------------------------------------- | --------------------------------------- |
| 核心数据飙升 | 事件相关核心数据（星标/确诊数/政策提及数/产品下载量） | 单日环比增长 300%+ 或连续3天增长 ≥ 100% |
| 数据稀缺性   | 该类数据此前长期平稳，无大幅波动记录                  | 近3个月内数据波动幅度 ≤ 20%             |
| 数据覆盖范围 | 核心数据从"单点"向"多点"扩散                          | 数据覆盖主体数 24小时内从1个增至3个+    |

#### 3. 主体属性信号

重大事件早期，权威主体、国家机构、头部企业会出现非常规动作。

| 信号类型       | 量化/质性指标                                      | 基础阈值                                |
| -------------- | -------------------------------------------------- | --------------------------------------- |
| 权威主体介入   | 政府部门、行业协会、国际组织发布相关公告/提醒/调研 | 出现 1个及以上权威主体动作              |
| 头部主体跟进   | 行业头部企业/机构对事件进行布局/研究/回应          | 3家及以上头部主体同步动作               |
| 主体动作紧急性 | 权威/头部主体的动作为"临时加更""紧急发布"          | 动作发布时间为非工作时间（深夜/节假日） |

#### 4. 内容特征信号

早期萌芽内容虽简短，但包含"新现象/新问题/新技术"核心信息。

| 信号类型   | 量化/质性指标                                        | 基础阈值                   |
| ---------- | ---------------------------------------------------- | -------------------------- |
| 内容创新性 | 内容包含"首次出现""全新""突破"等创新关键词           | 出现 1个及以上创新关键词   |
| 问题严重性 | 内容涉及"生命安全""重大财产损失""社会稳定"等严重问题 | 出现 1个及以上严重性关键词 |
| 可传播性   | 内容具备"故事性""争议性""情感共鸣"等传播要素         | 情感分析得分 ≥ 0.7         |

#### 5. 行业反馈信号

早期事件在行业内引发讨论、争议、快速跟进。

| 信号类型     | 量化/质性指标                | 基础阈值                       |
| ------------ | ---------------------------- | ------------------------------ |
| 行业讨论热度 | 行业内专业社群/论坛的讨论量  | 24小时内讨论量环比增长 200%+   |
| 行业争议性   | 行业内出现明显分歧观点       | 正反观点比例在 3:7 至 7:3 之间 |
| 快速跟进     | 行业内出现模仿/复制/对标动作 | 48小时内出现 2个及以上跟进动作 |

#### 6. 小众趋势信号

早期事件在小众群体中形成趋势，具备向大众扩散的潜力。

| 信号类型     | 量化/质性指标                | 基础阈值                         |
| ------------ | ---------------------------- | -------------------------------- |
| 小众群体热度 | 小众社群/圈子的内容互动量    | 互动量超过该社群历史均值 5倍+    |
| 跨圈层传播   | 内容从小众圈子向关联圈子扩散 | 24小时内扩散至 2个及以上关联圈子 |
| KOL 关注     | 领域内意见领袖开始关注/转发  | 出现 1个及以上 KOL 关注          |

## 新闻评判标准

### 核心原则

以「对人们生活核心领域的影响维度 + 影响程度」作为新闻重要性的唯一评判标准。单条新闻可标注多个影响领域，按「核心影响领域 + 次级影响领域」区分，优先依据核心影响的深度、广度、持续性判定是否为"现象级新闻"。

## 技术架构

### 前端

- 移动端优先适配
- 近三天大新闻模块（默认展示，按日期倒序）
- 历史归档模块（标题≤8字 + 日期，按时间倒序）
- 观察池/候选池管理界面（仅管理员可见）

### 后端

- 实时监测六大维度萌芽信号
- 定时调用新闻挖掘模型获取数据
- 三级事件池自动升级/降级逻辑
- 提供新闻数据的增删改查接口
- 权限管理：区分普通用户（只读）和管理员（可编辑）

### 数据库

存储新闻数据，字段包括：

- 标题
- 内容
- 日期
- 是否认可
- 来源 URL
- 核心影响领域
- 次级影响领域
- 事件池等级（观察池/候选池/确定池）
- 萌芽信号得分
- 创建时间
- 更新时间

## 数据管理规则

### 初始数据

- 预填充近一年的历史大新闻
- 每月至少有一条新闻

### 定时更新

- 观察池：每小时监测一次
- 候选池：每30分钟监测一次
- 确定池：每天北京时间 8:00 更新
- 近三天必须至少有一条新闻
- 超过三天的新闻自动归档

## 用户交互与反馈机制

### 权限说明

- **普通用户**：仅可查看确定池中的新闻内容
- **管理员**：可查看所有事件池，对新闻进行认可/不认可/锁定操作

### 认可/不认可流程

1. **认可**：打勾标记，新闻保留
2. **锁定**：标记为"高潜力"，锁定在候选池，不受信号衰减影响
3. **不认可**：
   - 后端立即重新查找当月新闻，或近三天新闻
   - 前端提示"正在重新查找新闻"
   - 找到后更新数据库并弹窗提示
   - 页面自动刷新展示新新闻
4. **循环直到认可**：历史新闻中每月至少有一条被认可的新闻，近三天至少有一条被认可的新闻

### 模型优化

根据管理员的反馈数据训练模型，持续提升新闻早期识别的准确性。

## 环境配置

### 后端配置 (backend/.env)

```env
# 数据库配置
DATABASE_URL=mysql+pymysql://root@localhost:3306/news_digger

# JWT 密钥
SECRET_KEY=your-secret-key-change-in-production

# 本地大模型配置 (Ollama + Qwen2.5-1.5B)
LLM_PROVIDER=ollama
LLM_MODEL=qwen2.5:1.5b
LLM_BASE_URL=http://localhost:11434

# 新闻数据源 API (可选)
NEWSAPI_KEY=your-newsapi-key
TIANAPI_KEY=your-tianapi-key
```

### 安装 Ollama 并下载模型

```bash
# 1. 安装 Ollama (macOS)
brew install ollama

# 2. 启动 Ollama
ollama serve

# 3. 下载千问 1.5B 模型
ollama pull qwen1.5:1.5b
```

### 启动命令

```bash
# 1. 启动后端
cd backend
source venv/bin/activate
uvicorn app.main:app --reload --port 8000

# 2. 启动前端
cd frontend
npm run dev
```
