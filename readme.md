# 重大新闻挖掘器

一个智能新闻筛选系统，自动识别并推送具有**重大影响力**的新闻，帮助用户高效获取关键资讯。

## 项目背景

面对海量新闻资讯，用户往往需要花费大量时间筛选。本项目旨在通过 AI 自动识别真正有影响力的"现象级"新闻，提升信息获取效率。

## 核心功能

- **智能筛选** — AI 大模型自动识别重大新闻并输出梗概
- **时效展示** — 优先展示近三天的大新闻
- **历史归档** — 归档历史新闻，便于回顾查阅

## 新闻评判标准

### 核心原则

以「对人们生活核心领域的影响维度 + 影响程度」作为新闻重要性的唯一评判标准。单条新闻可标注多个影响领域，按「核心影响领域 + 次级影响领域」区分，优先依据核心影响的深度、广度、持续性判定是否为"现象级新闻"。

### 跨领域影响判定

针对单事件辐射多个领域的情况，按「核心影响优先 + 影响程度叠加」原则判定，避免因单一类别局限而遗漏重要新闻。

## 新闻源

1. 社交媒体的热搜（抖音、微博、推特、知乎、小红书、B站、推特、Youtube）
2. RSS（新华社、央视网、人民网、澎湃新闻、南方周末、凤凰网、财新网、BBC、路透社、纽约时报、CNN）等等国内外知名新闻RSS源
3. 垂直社区（豆瓣、丁香园、雪球、36氪 / 虎嗅 / 少数派 / github）

注意：这些新闻源我全都要，一个都不能少，你要找到对应的方法去获取相应平台上的数据。

## 技术架构

### 前端

注意：前端托管到github pages，但我原先有一个blog项目已经在用github pages了(https://liuqinh2s.github.io/blog/)，不知道能不能两个项目一起用，如果能的话，链接就是：https://liuqinh2s.github.io/news/

- 移动端优先适配
- 近三天大新闻模块（默认展示，按日期倒序）
- 历史归档模块（标题 ≤ 8 字 + 日期，按时间倒序）
- 双主题支持（亮色/暗色），用户可自由切换
  - 亮色参考：https://lucent-klepon-404cd3.netlify.app/
  - 暗色参考：https://merry-dusk-4416df.netlify.app/

### 后端

不用后端。

GitHub Actions 充当"无服务器后端"，定时跑爬虫 + LLM 生成内容，然后把结果作为静态文件提交到仓库，前端只需要读取 site/data/ 下的 JSON 和 Markdown 文件就行了。API Key 都存在 GitHub Secrets 里（FIRECRAWL_API_KEY、OPENAI_API_KEY 等），不会暴露在代码中。

### 数据库

不用数据库，直接就跟写博客一样，静态网站，github仓库本身就是数据库。
