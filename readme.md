# 新闻挖掘器

一个智能新闻筛选系统，自动识别并推送具有重大影响力的新闻，帮助用户高效获取关键资讯。

## 项目背景

面对海量新闻资讯，用户往往需要花费大量时间筛选。本项目旨在通过 AI 自动识别真正有影响力的"现象级"新闻，提升信息获取效率。

## 核心功能

- **智能筛选** — AI 大模型自动识别重大新闻并输出梗概
- **时效展示** — 优先展示近三天的大新闻
- **历史归档** — 归档历史新闻，便于回顾查阅
- **人机协同** — 通过用户反馈持续优化筛选模型

## 技术架构

### 前端

- 移动端优先适配
- 近三天大新闻模块（默认展示，按日期倒序）
- 历史归档模块（标题 ≤ 8 字 + 日期，按时间倒序）
- 双主题支持（亮色/暗色），用户可自由切换
  - 亮色参考：https://lucent-klepon-404cd3.netlify.app/
  - 暗色参考：https://merry-dusk-4416df.netlify.app/

### 后端

- 获取新闻数据并传递给新闻挖掘模型（AI 大模型）
- 提供新闻数据的增删改查接口
- 权限管理：普通用户（只读）/ 管理员（可编辑）

### 数据库

| 字段        | 说明           |
| ----------- | -------------- |
| 标题        | 新闻标题       |
| 内容        | 新闻正文       |
| 日期        | 发布日期       |

## 新闻评判标准

### 核心原则

以「对人们生活核心领域的影响维度 + 影响程度」作为新闻重要性的唯一评判标准。单条新闻可标注多个影响领域，按「核心影响领域 + 次级影响领域」区分，优先依据核心影响的深度、广度、持续性判定是否为"现象级新闻"。

### 跨领域影响判定

针对单事件辐射多个领域的情况，按「核心影响优先 + 影响程度叠加」原则判定，避免因单一类别局限而遗漏重要新闻。

## 数据管理规则

### 初始数据

- 预填充近一年的历史大新闻
- 每月至少保留一条新闻

### 实时监控

- 近三天至少有一条新闻
- 超过三天的新闻自动归档

## 用户交互与反馈机制

### 权限说明

| 角色     | 权限                        |
| -------- | --------------------------- |
| 普通用户 | 仅可查看新闻内容            |
| 管理员   | 可对新闻进行认可/不认可操作 |

### 认可/不认可流程

1. **认可** — 打勾标记，新闻保留
2. **不认可**：
   - 后端立即重新查找当月新闻（归档中）或近三天新闻（近三天）
   - 前端提示"正在重新查找新闻"
   - 找到后更新数据库并弹窗提示"已重新找到一条新闻"
   - 页面自动刷新展示新新闻
3. **循环直到认可** — 历史新闻每月至少一条被认可，近三天至少一条被认可

### 模型优化

根据管理员的反馈数据训练 AI 大模型，持续提升新闻筛选的准确性。
