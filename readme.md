# 新闻挖掘器

一个智能新闻筛选系统，自动识别并推送具有重大影响力的新闻，帮助提高获取新闻的效率。

## 项目背景

不想花时间浏览大量不太重要的新闻，希望系统能自动筛选出真正有影响力的"现象级"新闻。本项目旨在实现这一目标。

## 核心功能

- **智能筛选**：自动识别具有重大影响力的新闻
- **时效展示**：优先展示近三天的大新闻
- **历史归档**：归档历史大新闻，便于回顾
- **人机协同**：通过用户反馈持续优化筛选模型

## 技术架构

### 前端

- 移动端优先适配
- 近三天大新闻模块（默认展示，按日期倒序）
- 历史归档模块（标题≤8字 + 日期，按时间倒序）

### 后端

- 定时调用新闻挖掘模型获取数据
- 提供新闻数据的增删改查接口
- 权限管理：区分普通用户（只读）和管理员（可编辑）

### 数据库

存储新闻数据，字段包括：

- 标题
- 内容
- 日期
- 是否认可
- 来源 URL
- 核心影响领域
- 次级影响领域
- 创建时间
- 更新时间

## 新闻评判标准

### 核心原则

以「对人们生活核心领域的影响维度 + 影响程度」作为新闻重要性的唯一评判标准。单条新闻可标注多个影响领域，按「核心影响领域 + 次级影响领域」区分，优先依据核心影响的深度、广度、持续性判定是否为"现象级新闻"。

## 数据管理规则

### 初始数据

- 预填充近一年的历史大新闻
- 每月至少有一条新闻

### 定时更新

- 每天北京时间 8:00 更新
- 近三天必须至少有一条新闻
- 超过三天的新闻自动归档

## 用户交互与反馈机制

### 权限说明

- **普通用户**：仅可查看新闻内容
- **管理员**：可对新闻进行认可/不认可操作

### 认可/不认可流程

1. **认可**：打勾标记，新闻保留
2. **不认可**：
   - 后端立即重新查找当月新闻，或近三天新闻
   - 前端提示"正在重新查找新闻"
   - 找到后更新数据库并弹窗提示
   - 页面自动刷新展示新新闻
3. **循环直到认可**：历史新闻中每月至少有一条被认可的新闻，近三天至少有一条被认可的新闻

### 模型优化

根据管理员的反馈数据训练模型，持续提升新闻筛选的准确性。

## 技术选型

- [ LLM（本地部署 Ollama + Qwen） ] 新闻挖掘模型选型

- [ React ] 前端技术栈
- [ Python3 + FastAPI ] 后端技术栈
- [ MySQL ] 数据库
- [ OAuth（Google/Apple） ] 用户认证
- [ APScheduler ] 定时任务管理新闻更新
